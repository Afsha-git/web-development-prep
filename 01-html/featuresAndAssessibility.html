1. HTML5 Semantic Elements

Why Semantic HTML?
  • ✅ Better SEO (search engines understand structure)
  • ✅ Improved accessibility (screen readers work better)
  • ✅ Easier to read and maintain code
  • ✅ More meaningful than generic divs

Main Semantic Tags:

Tag Purpose Example Use
<header> Header section Site header, article header
<nav> Navigation links Main menu, breadcrumbs
<main> Main content (unique per page) Primary page content
<article> Independent content Blog post, news article
<section> Thematic grouping Chapters, topics
<aside> Sidebar/related content Ads, related links
<footer> Footer section Copyright, contact info
<figure> Self-contained content Image with caption
<figcaption> Caption for figure Image description
<time> Date/time Publication date
<mark> Highlighted text Search results highlight

Example Structure:

<body>
  <header>
    <nav>Navigation</nav>
  </header>
  
  <main>
    <article>
      <section>Content</section>
    </article>
    
    <aside>Sidebar</aside>
  </main>
  
  <footer>Footer</footer>
</body>

5. Custom Data Attributes

Data attributes let you store custom data on HTML elements.

Syntax:

<div data-user-id="12345" data-role="admin" data-status="active">
  User Profile
</div>

  • Keep metadata without affecting display

6. Web Accessibility (ARIA)

Why Accessibility Matters:
  • ✅ Helps people with disabilities use your site
  • ✅ Required by law in many countries
  • ✅ Improves SEO
  • ✅ Better user experience for everyone

ARIA Roles:

Role Purpose Example
role="banner" Site header <header role="banner">
role="navigation" Navigation menu <nav role="navigation">
role="main" Main content <main role="main">
role="contentinfo" Footer info <footer role="contentinfo">
role="search" Search section <form role="search">
role="button" Custom button <div role="button">
role="alert" Alert message <div role="alert">

ARIA Attributes:

Attribute Purpose
aria-label Accessible label for element
aria-labelledby ID of labeling element
aria-describedby ID of describing element
aria-hidden Hide from screen readers
aria-live Announce dynamic changes
aria-required Mark field as required

Examples:

<button aria-label="Close dialog">X</button>

<img src="logo.png" alt="Company Logo" aria-hidden="true">

<div role="alert" aria-live="polite">
  Form submitted successfully!
</div>

<input type="text" aria-required="true" aria-describedby="help-text">
<small id="help-text">Enter your full name</small>

7. Accessibility Best Practices

✅ DO:

  • Use semantic HTML - Use proper tags (<nav>, <main>, etc.)
  • Alt text for images - Always provide descriptive alt text
  • Label form inputs - Every input needs a <label>
  • Keyboard navigation - Ensure all features work with keyboard
  • Color contrast - Text must be readable (WCAG standards)
  • Focus indicators - Show where keyboard focus is
  • Heading hierarchy - Use h1-h6 in order
  • Skip links - Allow skipping to main content
  • Language attribute - <html lang="en">

❌ DON'T:

  • Don't use divs for buttons - Use <button>
  • Don't skip heading levels - h1 → h3 is wrong
  • Don't use color alone - Add text/icons too
  • Don't use "click here" - Be descriptive
  • Don't forget keyboard users - Test with Tab key
  • Don't auto-play audio/video - Let users control
  • Don't use empty alt text - Unless decorative

Proper Alt Text Examples:

Image Type Good Alt Text Bad Alt Text
Logo Company Name logo Image, Logo
Informative Bar chart showing sales growth Chart, Graph
Decorative alt="" (empty) Decoration
Functional Search button Button
